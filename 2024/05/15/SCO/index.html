當龐克輯錄成為學術中的一環？

這篇會跟大家分享自學這件事
幾個關鍵字如下：

  1. （元）啟發式演算法 (Metaheuristics)
  2. 社會認知算法 (Social Cognitive Optimization, SCO)

先來說一下我跟它的初識：因為宗哲老師這學期【啟發式演算法的解題與應用】課程期末專題需求，我們這組選了一個「使用元啟發式演算法設計對稱式加密系統」的題目，原 paper 使用常見的模擬退火法 (Simulated annealing, SA) 和基因演算法 (Genetic algorithm, GA) 實作，如果繼續調整參數做實驗似乎感覺不會得到更好的結果（畢竟人家都提出來這樣了，肯定是經過實驗過的，而且前幾次作業很明顯的告訴我們對於調整參數的技能點數是 0，所以無職轉生成調參工程師還是先等等），於是我們打算三個人一人使用一套沒聽過的演算法來實驗能不能做得更好，至少還能學到其他啟發式演算法，之後跟大家分享也不錯！

元啟發式演算法常見的就是把各種動物抓過來研究，比方說人工蜂群演算法 (Artificial bee colony, ABC)、蟻群演算法 (Ant colony optimization, ACO) 都很常見，有了這些基礎之後學界就慢慢誕生其他各式各樣的演算法，比如說飛蛾、鯊魚、蚯蚓、帝王企鵝、抹香鯨算法等等，會發現動物明開始聚焦了

正當選擇障礙症出現的時候突然在維基突然瞄到兩個看似很有趣又與眾不同的演算法，一個是人類精神搜索算法 (Human mental search, HMS)，另一個就是社會認知算法 (Social Cognitive Optimization, SCO)，立刻吸引我的興趣，其實我是對 HMS 比較感興趣的，但了解之後發現 HMS 是建構在「競標」的場景，裡面也用到很多暫時還不會的東西，例如 Levy flight，所以就先跟它說掰掰，撿起第二喜歡的 SCO

有修教程的不要害怕，整篇不會出現皮亞傑，你的擔憂我也曾經想過，但 SCO 沒有！

# 什麼是社會認知算法

所以什麼是 SCO？這部分我覺得最近的航海王說明得很好

<圖還在找>

從漫畫中可以看到：貝加龐克博士的分靈體（抱歉錯棚了，是分身）雖然彼此獨立，但因為每天都會把記憶、學習到的東西放進龐克輯錄，所以隔天所有分身都能拿到這份資料，這樣即便沒有親身經歷過，別人的經驗也會變成我的經驗，是一個很逆天的技能！按照這個想法，如果我也創造九個分身，然後現在開始跟分身們努力工作一年，明年就可以說「我是擁有十年資歷的資深工程師」

在正式介紹 SCO 之前先來介紹一些專有名詞：

  1. 知識點 (knowledge point)：解空間中的一個解就是一個知識點，當然會有對應的知識含量 (fitness value)，可以想成圖書館中的每一本書
  2. 知識集 (library)：一群知識點構成的群體，可以想成龐克輯錄或是圖書館
  3. 主體 (learning agent)：一般人，但每個人手上都可以握有一個在知識集的知識點，這邊要注意主體不是一個知識點，而是握有知識點的容器（說容器怪怪的就是了）

最後，SCO 的故事是這樣的：在這個世界上有很多知識點，一開始我們先組成 N_c 人的研究小組，要在 T 個週期內盡量從知識集和我們的研究中找出含金量最高的知識點，概念很簡單，接著就來看這套機制會經歷那些事件。

# SCO 流程

這個章節來說明 SCO 的步驟，用個清單來看會長這樣：

  • 初始化
  • 學習週期
    • 主體認知
      • 榜樣選擇
      • 質量衡量
      • 社會學習
      • 知識反饋
      • 知識更新
    • 知識集維護
  • 結束

# 初始化 Initiaization

首先我們要初始化，初始化分成兩部分

一是要初始化圖書館，我們必須把圖書館的館藏建立好，並標記好它的知識含量，這邊假設館藏有 N_{pop} 本
二是要初始化主體，我們要讓每個人手上都握有一個知識點，代表他們目前會的東西

# 學習週期 Learning Cycles

接著是啟發式演算法都有共通的假設：資源有限，所以在有限的資源內我能找到多好的解。以 SCO 而言，它會定義學習週期為 T，經過 T 個週期之後就要回傳最好的知識點。當然這邊其實就是一個 stopping criteria，如果想用其他例如 times of fitness calls 也沒問題

# 主體認知 Victarious Learning

這邊開始描述每一個主體 x_i (t) 每一輪週期會做的事情，也就是說這個 learning agent 到底是怎麼學習的

# 榜樣選擇 Model Selection

首先是榜樣選擇，他會去 library 撈 TAU_B 本書出來看，從中選擇最好的（高質量知識點）那本 model 做為榜樣

# 質量衡量 Quality Evaluation

再來要去衡量主體跟榜樣誰比較好，比較好的那個我們叫他 base，會是等等做研究的基礎，另一個比較差的就會是 ref，也就是拿來參考而已

# 社會學習 Learning

學習就是在做研究，會無中生有產生一個新的知識點 x_i (t+1)，這個新解會透過 base 和 ref 得到，我們會以 base 作為基準，看 ref 跟 base 的差異多大決定要產生怎麼樣的新解

# 知識反饋 Knowledge Sharing

新的知識點 aka 研究成果就應該提交至 conference 或 journal 被收錄在圖書館館藏中，這邊也不例外，所以我們會把新知識點放進圖書館

# 知識更新 Knowledge Update

再來因為主體都研究出一個新知識了，沒道理拿著就知識不放手，所以我們會把主體手上拿的知識點 x_i (t) 換成新知識點 x_i (t+1)，注意這邊就知識點還在，因為它存在於圖書館中，只是主體手上拿的變其他知識而已

# 知識集維護 Library Refreshment

在每個主體經過一個週期 t 都做了一番研究後，圖書館館藏多了 N_c 本，可是圖書館大小有限，所以最後我們會對館藏進行篩選，把多出來的那些館藏藏淘汰掉，就像圖書館會把舊書拿出去送人。這邊可以注意淘汰的書一定是比較沒有價值的，也就是知識含量較低的知識點

# 結束 Termination

最後最後，在所有學習歷程都結束之後，就可以回傳最好的知識點是誰了

# 總結

那麼 SCO 到底可以看過多少解呢？答案是 N_{pop} + T * N_c，所以會發現研究小組有多少人在 T 比較長的時候反而是重點，跟 GA 某種程度來說差不多

對 SCO 有興趣的人可以去 Reference 找原論文，這邊只是用很簡單的想法介紹 SCO 是什麼，當然內容還會再更新，畢竟這是深夜寫出來的東西。。
